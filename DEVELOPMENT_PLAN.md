# 영단어 스피드 퀴즈 개발 계획 (모바일 웹)

## 1) 목표와 범위
- 모바일 우선(Responsive) 웹페이지 구축
- 제목: **영단어 스피드 퀴즈**
- 닉네임 입력 후 시작
- 난이도: **초 / 중 / 고** (TOEIC 중심 단어)
- 문제 구성: 이미지 1장 + 4지선다(정답 1, 오답 3)
- 총 제한 시간: **2분**
- 타이머: 다이너마이트 심지 길이로 시간 표현 + 심지가 타들어가는 애니메이션

## 2) 핵심 UX 흐름
1. 랜딩 화면
   - 타이틀, 닉네임 입력, 난이도 선택, 시작 버튼
2. 퀴즈 화면
   - 상단: 남은 시간(다이너마이트 심지 애니메이션 + 숫자)
   - 중앙: 이미지
   - 하단: 4지선다 버튼
3. 결과 화면
   - 정답 수, 오답 수, 정확도
   - 다시 시작 버튼

## 3) 기술 스택(가정)
- HTML / CSS / Vanilla JS
- 모바일 퍼스트 레이아웃
- 빌드 도구 없이 정적 페이지로 먼저 구현

## 4) 이미지 API 선택
### 후보
- Pexels API
- Pixabay API

### 선택
- **Pexels API 우선 적용**
  - 무료 사용 가능
  - API 키 발급이 간단하고 문서가 잘 정리됨
  - 사용 시 **출처 표기 요구**

## 5) 데이터 설계
### 단어 데이터(TOEIC 중심)
- 난이도별 단어 목록 JSON 구성
- 각 항목: 
  - `id`, `word`, `meaning`, `difficulty`, `imageQuery`

### 문제 생성 로직
- 난이도별 랜덤 셔플
- 각 문제마다:
  - 정답 단어 1개
  - 같은 난이도에서 오답 3개 샘플링

## 6) 타이머/애니메이션
- 총 시간: 120초
- 1초 단위 카운트다운
- 심지 길이 = 남은 시간 비율
- CSS 애니메이션 + JS로 진행률 동기화

## 7) 화면 설계(요약)
- **랜딩 화면**: 타이틀, 닉네임 입력, 난이도 선택 버튼, 시작 버튼
- **퀴즈 화면**: 타이머, 이미지, 선택지 버튼 4개
- **결과 화면**: 점수 요약, 재시작

## 8) 이미지 API 연동 흐름
1. `imageQuery`로 API 검색
2. 이미지 1장 랜덤 선택
3. 이미지를 문제 카드에 표시
4. 출처 표기 영역에 제공자/작가 표시

## 9) 개발 단계
1. **UI 스캐폴딩**: 3화면 구성, 반응형
2. **퀴즈 로직**: 데이터/난이도/정답 검증
3. **타이머 애니메이션**: 심지 길이 + 불꽃 효과
4. **이미지 API 연동**: 검색, 로딩 상태, 에러 처리
5. **결과 화면**: 점수/재시작
6. **테스트**: 모바일 실기기, 다양한 해상도

## 10) 리스크 및 대응
- 이미지 API 속도/제한
  - 로딩 스켈레톤, 실패 시 로컬 플레이스홀더 이미지
- API 키 노출
  - 배포 시 프록시 서버 또는 환경변수 주입 고려

## 11) 산출물
- `index.html`
- `styles.css`
- `app.js`
- `data/words.json`
- `assets/` (타이머 불꽃, 기본 이미지 등)

## 12) 오픈 이슈(확인 필요)
- 최종 선택 이미지 API 확정 (Pexels vs Pixabay)
- 단어 난이도 기준 수량
